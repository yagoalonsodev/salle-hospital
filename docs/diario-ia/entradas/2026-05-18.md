### Entrada 004 — MongoDB centralizada para todos los logs

| Campo | Valor |
|-------|-------|
| Fecha | 2026-05-18 |
| Herramienta | Cursor |
| Resultado | corrección |
| Spec SDD | D8M-01 (logging centralizado ampliado) |

#### Prompt del usuario

```
Créame una MongoDB y guárdame todos los logs. Las contraseñas y credenciales pon las
mismas que las del .env. Hazlo tú todo.
```

#### Respuesta de la IA (resumen)

1. **Servicio `mongodb`** en `docker-compose.yml` (puerto 27017, volumen `mongo_data`, init `infra/mongodb/init-mongo.js`).
2. **Colecciones** en BD `salle_logs`: `application_logs`, `airflow_logs`, `file_logs`, `log_sync_cursors`.
3. **`scripts/mongo_log_store.py`**: handler `MongoLogHandler` + `attach_mongo_handler()`; integrado en API Flask, ML FastAPI, watcher, pipeline y `salle_logging.py`.
4. **`salle-log-sync`**: daemon que sincroniza `airflow/logs/` y `logs/` (p. ej. `verify_stack.log`) cada 60 s.
5. **`scripts/query_logs.py`**: consulta CLI desde el contenedor log-sync.
6. Variables en `.env.example`: `MONGO_*` y `MONGODB_URI` (mismas credenciales que `POSTGRES_*` en desarrollo).

#### Corrección

| Error | Solución |
|-------|----------|
| MongoDB **unhealthy** (healthcheck) | Comillas y `$$` correctos en `mongosh` del healthcheck |
| **Authentication failed** en log-sync | URI con `}` extra en compose (`salle_secret}@`) — una llave de más en interpolación |
| API/ML **Exited (3)** | `import sys` dentro del `try` en `logging_config.py` → `UnboundLocalError` en `sys.stdout` |

#### Archivos afectados

- `docker-compose.yml`, `.env.example`, `infra/mongodb/`, `infra/log-sync/`
- `scripts/mongo_log_store.py`, `log_sync_daemon.py`, `query_logs.py`, `salle_logging.py`, `env_utils.py`
- `api/app/logging_config.py`, `ml/app/logging_config.py`, `requirements*.txt`
- `README.md`, `docs/specs/BACKLOG.md`

#### Reflexión

Centralizar logs en MongoDB encaja con el Día 8 (monitorización) y facilita auditoría en demo. Los fallos fueron de compose (interpolación) y de Python (sombreado de `sys`), no del motor Mongo. No documentar contraseñas en el diario: solo referencia a `.env.example`.
